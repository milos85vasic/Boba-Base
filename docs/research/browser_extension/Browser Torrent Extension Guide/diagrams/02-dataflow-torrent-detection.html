Flow Description

Phase ① Detect: Content Script observes DOM changes via MutationObserver, scans for magnet: links, .torrent file URLs, and bare infohashes.

Phase ② Parse: Extracted links are normalized by the parser module — magnet URIs decoded, .torrent URLs validated, infohashes formatted.

Phase ③ Route: Parsed torrents are sent to Background SW which manages the send queue, handles auth, and routes to BobaAPIClient.

Phase ④ Validate: Boba Services (FastAPI or Go) validate torrent data, perform metadata lookup, and prepare the add request.

Phase ⑤ Download: qBitTorrent WebUI receives the add request, BitTorrent engine connects to peers, downloads pieces, verifies hashes.

Phase ⑥ Complete: Downloaded files saved to /downloads/complete/, response propagated back through the chain, UI updated.
